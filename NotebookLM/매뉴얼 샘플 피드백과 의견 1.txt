매뉴얼 샘플 피드백

올려주신 샘플에 대한 피드백입니다.

형식 부분
1. 가이드라인의 지침이 구체적이지 않아 집필 시 형식과 내용을 구성하기 위해서는 샘플을 여러 차례 읽어봐야 가이드라인의 내용이 이해가 될 것으로 보임. 가이드라인이 좀더 구체화되어야할 것으로 보임.
2. 번호 체계가 가이드라인에 명확히 제시되지 않아 샘플을 보고 파악해야 하는 번거로움이 있음. 또한, 번호 체계가 대분류는 번호+마침표 중분류는 번호-번호+마침표로 구성되고, 실행 순서의 절차가 역시 대분류와 같은 번호+마침표 번호 체계로 구성되어 있어 대분류의 번호 체계와 실행 순서의 번호 체계가 중복됨. 실행 순서의 절차를 나타내는 번호 체계는 동그라미 숫자를 사용할 것을 제안함.
3. 집필 서식 틀 차원에서 위에서 피드백한 것과 같이 샘플을 여러 차례 읽어야 집필 서식의 번호 체계를 비롯한 틀을 이해할 수 있으므로, 샘플과 별도로 집필 서식 틀을 가이드라인에 제시해줄 것을 제안함.4. Q&A를 샘플에서 번호 체계상 대분류 번호+마침표로 부여해서 작성했는데, 이렇게 집필하면 AI툴의 기능과 Q&A가 같ㅇㅡㄴ 층위의 항목으로 구성되어 체계상 적절하지 않아 보임. Q&A는 번호 체계를 부여하지 않고 글상잘로 제시해주는 것을ㄹ 제안함.

내용 부분
1. 샘플의 실행 순서 부분의 내용이 스크린리더 사용자가 보면서 바로 따라하기에 너무 지나치게 간결하게 작성되어 있는 것으로 보임. 아주 간결한 실행 절차만을 원하는 사용자도 있는가 하면 상대적으로 구체적인 실행 절차에 대한 가이드를 원하는 사용자도 있어 이 부분은 어느 정도 논의를 통한 의사결정이 필요해 보임.
2. 샘플의 화면 구성 하위 항목에 버튼: “소스 업로드”와 같은 것은 스크린리더가 ‘소스 업로드 버튼’과 같이 버튼을 뒤에서 읽으므로 스크린리더가 실제로 읽는 방식으로 수정 필요.
3. 샘플의 내용 중 스크린리더가 읽는 대체텍스트와 다르게 실행 순서에 설명이 있는 것으로 보임. 이것은 아무래도 ㅐㅁ플 초안을 NotebookLM으로 작성하고 검수 작업이나 다듬는 과정을 거치지 않은 것으로 추측됨. 내용에 대한 어느 정도 다듬는 과정이 필요해 보임.

편도환 선생님 피드백에 대한 의견입니다.

1. 집필 가이드 구체성과 템플릿 제공
- 설명 번호 체계 포함하여 보강하고, 템플릿 제공하겠음.

2. 번호 체계 관련
- 실행 순서 섹션에서 1., 2., 3. ...으로 표시한 것은 추후 웹 페이지로 올라갈 때 그렇게 해야 html 요소 중 '순서 있는 목록(ol)'으로 처리되기 때문임. 목록으로 처리되면 센스리더 탐색이 단축키(I 키)로 조금 더 용이함.
- 만약 순서 있는 목록(ol) 처리를 포기한다면 대신 제안하는 것은 동그라미 번호보다는 Step 1., Step 2., Step 3. ...을 제안함. 이유는 동그라미 번호는 센스리더 읽기 기본값에서 읽지 않게 설정되어 있기 때문. LLM도 동그라미 번호를 잘 처리하지 못함.
- 결론적으로, 웹 페이지에 구현될 때 순서 있는 목록(ol)로 처리할지, 일반 텍스트로 처리할지 의견 구함.

3. 실행 순서 구체화
- 실행 순서를 구체화할 경우, 두 가지 방법이 있음. 현행 간략한 단계 소개 아래 단락을 한두 개 추가하여 글로 풀어쓰는 방법, 또는 현행 실행 순서에서 단계별 설명을 조금 더 자세히 쓰는 방법임.
- 그런데 이 경우 집필위원들의 업무가 크게 증가함. NotebookLM은 비교적 단순한 구성임에도 세부 기능이 15가지로 나왔는데 각각의 설명을 보강한다고 할 때 전체적 분량이 크게 늘어날 것으로 예상됨.
- 실행 순서를 얼마나 구체화할 것인지 의견 추가로 필요하다고 판단됨.

4. 센스리더 실제 출력 음성 반영
- 일부 검증한 것이 있고 검증하지 못한 것이 있는데 최종본에서는 검증하여 수정할 예정임.
- 버튼은 현재 "버튼: "추가" - 소스 추가" 식으로 다소 복잡하게 설명되어 있는데 간소화하겠음.

5. 자주 묻는 질문 섹션
- 이 부분은 AI로 생성한 섹션임.
- 다만, 실제 유용성이 얼마나 큰지 잘 모르겠음.
- 매뉴얼의 간소화와 집필위원 부담 경감 차원에서는 삭제해도 된다고 판단함.
- 만약 유지한다면 소개 섹션, 요약 섹션과 더불어 넘버링 제외하는 것이 일관성이 있어 보임. 박스 처리는 디자인적인 요소여서 웹 페이지 구현 방법은 잘 모르겠음.
- 자주 묻는 질문 섹션 유지 여부 의견 구함.
